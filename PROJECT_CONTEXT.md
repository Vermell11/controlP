# ControlP Context

ControlP is a local-first visual Agentic Project OS for Andres Ortega. It reads the Cerebro Obsidian vault as the project index, resolves local project folders under `/code`, and visualizes Git, Graphify, sessions, health, and next steps in a V.A.U.L.T.-style dashboard.

Current state: V1.5.3 delivered. The semantic/RAG boundary remains read-only and deterministic-first. Authorized intents use a canonical port backed by SQLite transactions, unique idempotency keys and an append-only event table; legacy JSONL is imported once with corruption accounting. Exact SHA-256 preview confirmation is required before the runner can claim a mutation with a lease. Schedule and project-card writes execute only through registered handlers and leave idempotent intent markers. Voice lifecycle recovers wake/PTT after TTS/errors. The model has no filesystem, write or credential access. Data contract `SCHEMA_VERSION 2`.

Navigation for new sessions: `CONTEXT_MAP.md` (repo root) maps every module — purpose, files, extension points, invariants, and a "to change X touch only Y" table; it is kept current at every session close. Architecture recipes live in Obsidian (`Proyectos/ControlP/Arquitectura/Routers y extensión`); a brief schematic manual with the Obsidian/Notion migration plan is in `docs/manual.md` (deliberately not in `public/`).

Architecture law: the ControlP Canonical Kernel owns domain data, identity, policies, events and versioned ports. Every database, agent, model, channel or external tool is a replaceable adapter with import/export, reconciliation and removal tests; no irreplaceable data may live only in a provider-native format.

Quality gates: `npm run check`, `npm test`, Gitleaks, online `npm audit` and a production Webpack build; registry enforces id/slug uniqueness. A root PostCSS override keeps Next on the audited PostCSS 8.5.17 release until its nested pin is updated upstream. Turbopack hangs in the iCloud folder, while Webpack completes — known environmental debt; move the repo out of iCloud. Mic requires localhost/https.
Reto actual: V1.5.4 identity, personal memory and vocabulary. Define canonical profile, permissions, conversation and vocabulary schemas on the existing operational SQLite boundary. All future mutations must reuse the mandatory `propuesta → vista previa → confirmación → ejecución → traza` pipeline. Roadmap in `Proyectos/ControlP/Roadmap`; data architecture in `Proyectos/ControlP/Arquitectura/Datos`.
