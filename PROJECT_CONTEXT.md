# ControlP Context

ControlP is a local-first visual Agentic Project OS for Andres Ortega. It reads the Cerebro Obsidian vault as the project index, resolves local project folders under `/code`, and visualizes Git, Graphify, sessions, health, and next steps in a V.A.U.L.T.-style dashboard.

Current state: consolidated and documented, ready for the LLM (V1.4.3). Voice runs on-device (V1.4.2: `stt/` sidecar behind the `/api/stt` proxy, swappable whisper.local ↔ web.speech providers, 16-band PTT synthesizer) and command execution is visually clean: the core's twinkle uses accumulated phase in the shader (never time × speed), so state-preset transitions never strobe regardless of uptime. Extension seams are clean and documented for Sprint 4: `routeCommand(transcript, projects)` may return `VoiceAction | Promise<VoiceAction>` — the caller already awaits, so the LLM router replaces `commands.ts` alone. Dead code removed; the Notion connector now composes activity titles (`{project} {version} — {activity}`) idempotently. All on the V1.3.1 base: three 3D formations, CameraDirector, operational HUD panels, editable records, `/queue`. Data contract `SCHEMA_VERSION 2`.

Navigation for new sessions: `CONTEXT_MAP.md` (repo root) maps every module — purpose, files, extension points, invariants, and a "to change X touch only Y" table; it is kept current at every session close. Architecture recipes live in Obsidian (`Proyectos/ControlP/Arquitectura/Routers y extensión`); a brief schematic manual with the Obsidian/Notion migration plan is in `docs/manual.md` (deliberately not in `public/`).

Quality gates (V1.2.2): `npm run check` (typecheck + explicit ESLint) is the official contract; registry enforces id/slug uniqueness. Note: `next build` hangs in the iCloud folder (`.next` indexing); it builds in ~5 s in a clean local copy — known environmental debt, recommendation is moving the repo out of iCloud. Mic requires secure context (localhost/https). Graphify pending regeneration (V1.4.3 closed without a graph chore).

Reto actual: Sprint 4 — LLM assistant (V1.5): an intelligent router behind the same `routeCommand` signature (async contract ready), RAG over capsules + Graphify, dispatch to Obsidian/Notion/Claude/Codex, and queue hardening before real execution. The operational agenda (Schedule as daily manager with voice capture) remains deferred and gets its version number when it enters construction. Roadmap in `Proyectos/ControlP/Roadmap`; data architecture in `Proyectos/ControlP/Arquitectura/Datos`.
